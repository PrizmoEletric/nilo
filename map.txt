# Nilo — Architecture Map

> Quick reference for AI agents and contributors. Read this before touching any module.

---

## What Nilo Is

Nilo is a Mineflayer-based Minecraft bot running on a **Fabric 1.20.1** server (Prominence II modpack). It has a persistent personality driven by a **Letta** memory agent (separate process on port 8283). Nilo responds to natural language in chat, executes behavior commands, and can autonomously learn and run skills generated by a local **Ollama** LLM.

**Entry point:** `node nilo.js`  
**Reconnects automatically** every 10s on disconnect.

## Module Map

```
nilo.js          Entry point. Creates bot, wires all events, boots log watcher.
├── state.js     Shared mutable state object. Single source of truth for runtime vars.
├── config.js    Constants + loadConfig/saveConfig (raw JSON file I/O only).
├── trust.js     Trust management: isTrusted, trustPlayer, untrustPlayer, listTrusted.
├── lang.js      detectLanguage(text) → 'en' | 'pt-BR'
├── letta.js     queryLetta(msg) → string. parseAction(raw) → {text, action}.
├── behavior.js  setBehavior(bot, mode, username). clearBehavior(bot).
├── movement.js  createMovements, installDoorOpener, startFollow, tryUnstuck.
├── items.js     Inventory helpers: getEquipDestination, isEquippable, pickupNearestItem, isBuildable.
├── combat.js    isHostileMob, weapon helpers, startAttack, startBowMode.
├── activities.js runFarm, collectGrave, startFishing, buildSimpleHouse, startDance, sleepInBed.
├── monitor.js   startProximityMonitor, startAutonomousBehaviors, startSkillAutonomyTicker, watchLog.
├── commands.js  handleNaturalCommand + all IS_* pattern matchers.
├── actions.js   dispatchAction (LLM action → function call). runCommand (chat_command packet).
├── skill-engine.js  Voyager-style skill system: learnSkill, runSkill, autonomousTick, curriculum.
└── skills/      Pre-built and LLM-generated skill JS files + manifest.json.
```

### Dependency order (no circular deps)

```
state ← config ← trust
                  lang
                  letta
      ← behavior ← movement ← items
                             ← combat
                             ← activities
                             ← monitor
                             ← commands ← actions
```

Each module only imports from modules above it in the chain. `nilo.js` imports everything.

## Shared State (`state.js`)

All runtime flags live in one exported object. Every module imports `./state` and reads/writes fields directly.

| Field | Type | Meaning |
|---|---|---|
| `activeBotRef` | Bot \| null | The live mineflayer bot instance |
| `behaviorMode` | string | Current behavior (see modes below) |
| `behaviorInterval` | timer \| {_cleanup} \| null | Active behavior ticker; `_cleanup()` for listener-based behaviors |
| `behaviorOwner` | string \| null | Username who issued the current behavior |
| `isFarming` | bool | Farm run in progress — blocks re-entry |
| `isLooting` | bool | Chest loot in progress — blocks explore |
| `exploringEnabled` | bool | Whether idle exploration is active |
| `proximityInterval` | timer | Handle for the proximity/health monitor |
| `autonomousInterval` | timer | Handle for the explore/look-around loop |
| `lastInteractionTime` | ms | Last time a player directly addressed Nilo |
| `justDied` | bool | Set on death, cleared after grave search |
| `autonomousSkillsEnabled` | bool | Whether the skill curriculum runs autonomously |
| `skillLearnInProgress` | bool | Prevents concurrent skill learn jobs |

---

## Behavior Modes

`state.behaviorMode` is always one of these strings:

| Mode | Description |
|---|---|
| `idle` | Default. Wanders if `exploringEnabled`. Responds to everything. |
| `follow` | Follows `behaviorOwner` using GoalFollow, 1s refresh loop. |
| `wander` | Sets a random GoalBlock every 5s. |
| `sit` | Sneak held true. Bot stays put. |
| `attack` | Pursues and attacks nearest hostile mob with melee. |
| `defensive` | Passive until hit — retaliates on `entityHurt`. |
| `passive` | Does not fight under any circumstance. |
| `fishing` | Cast/reel loop with bobber velocity detection. |
| `bow` | Ranged combat loop with kiting logic. |
| `building` | Block-placing loop for quick shelter. |
| `dance` | 50ms tick animation loop. |

`setBehavior(bot, mode, username)` always calls `clearBehavior` first, which kills any active `behaviorInterval` and clears pathfinder + control states.

## Chat Message Flow

```
bot.on('chat', username, message)
  │
  ├─ [MASTER only] handleNaturalCommand(bot, lower, message)
  │     Checks IS_* patterns in order. Returns true if matched → done.
  │     Physical actions (jump, move, sprint, spin, wave, crouch, look) handled here.
  │     Trust management handled here.
  │
  ├─ Determine if addressed to Nilo:
  │     - message contains "nilo" or starts with "!nilo"
  │     - OR: MASTER within 30s conversation window
  │     - OR: MASTER within 12 blocks
  │
  ├─ [MASTER only] Admin commands (!nilo quit, setfarm, setchest, farm,
  │     learn, do, skills, forget, queue, autonomous on/off,
  │     trust, untrust, trusted)
  │
  ├─ [Trusted non-MASTER] Behavior commands (!nilo follow/stay/sit/wander/attack/defensive/passive)
  │     Blocked if behaviorOwner === MASTER.
  │
  └─ Letta path:
        Strip trigger words → queryLetta(ctx) → parseAction(raw)
        → bot.chat(text)
        → dispatchAction(bot, action, username)  [if action present]
```

## Letta Integration (`letta.js`)

**`queryLetta(userMessage)`** — POST to LETTA_URL, returns the first `assistant_message` content string (emojis stripped).

**Context sent** always includes:
- Session hint (`[NEW SESSION]` if >5 min idle)
- Speaker name + message
- Inventory summary + held item
- Detected language (`en` or `pt-BR`)
- Available actions hint (see below)

**`parseAction(raw)`** — scans the end of the LLM reply for `[ACTION: name]`. Returns `{ text, action }`.

### LLM-dispatchable actions (`actions.js` → `dispatchAction`)

`follow` `stay` `stop` `sit` `come` `closer` `unstuck` `dance` `fish` `stop_fish` `bow` `build_house` `sleep` `wander` `attack` `defensive` `passive` `explore` `stop_explore` `collect_grave` `wave` `spin` `jump`

## Natural Language Commands (`commands.js`)

All handled **before** Letta. MASTER only (no prefix needed). Matched in order:

| Category | Examples |
|---|---|
| Movement | follow, come here, closer, stop following, stay, unstuck |
| Combat | attack, defensive, passive |
| Activities | fish, stop fishing, use bow, build shelter, sleep, dance, farm |
| Exploration | explore, stop exploring, wander |
| Physical | jump N times, move forward/back/left/right [N seconds], sprint, spin N times, wave, crouch, stand up, look up/down/north/south/east/west |
| Items | equip \<item\>, unequip, drop all, give \<item\>, eat this |
| Blocks | click/press/activate at X Y Z |
| Trust | trust \<name\>, untrust \<name\>, who do you trust |
| Meta | say \<text\>, repeat after me \<text\> |
| Grave | collect my grave, get my stuff |

Portuguese (pt-BR) variants exist for most commands.

## Trust System (`trust.js`)

Trust is persisted in `config.json` under the `trusted` array.

| Who | What they can do |
|---|---|
| MASTER (`PrizmoElectric`) | Everything. Always trusted. Cannot be untrusted. |
| Trusted players | `!nilo follow/stay/sit/wander/attack/defensive/passive`. Blocked if MASTER has locked behavior. |
| Everyone else | Can chat with Nilo (Letta responds), no commands. |

**MASTER manages trust via:**
- Chat: `trust Steve` / `untrust Steve` / `who do you trust?`
- Commands: `!nilo trust Steve` / `!nilo untrust Steve` / `!nilo trusted`

## Proactive Events (`monitor.js`)

Three sources of unprompted bot speech, all rate-limited and suppressed for 30s after login:

| Event | Trigger | Cooldown |
|---|---|---|
| Proximity greet | MASTER enters 15-block range | 2 min |
| Follow complaint | MASTER leaves 15-block range while following | 90s |
| Health warning | Bot health ≤ 8/20 | Once per low-health dip |
| Threat warning | Hostile mob within 16 blocks (idle/follow/wander only) | 20s |
| Log events | Player join/leave, deaths, advancements (from server log) | — |

**Session hint:** if `lastInteractionTime` was >5 min ago, Letta is told `[NEW SESSION]` to prevent memory bleed.

## Skill System (`skill-engine.js`)

Voyager-style: Ollama generates JS, a critic executes it, success → saved to disk.

**Files:**
- `skills/*.js` — each exports `async function niloSkill(bot)`
- `skills/manifest.json` — index of all skills with name, description, file, success/failure counts

**Commands (MASTER only):**

| Command | Effect |
|---|---|
| `!nilo learn <task>` | Generate + test a new skill (up to 3 retries) |
| `!nilo do <name>` | Run a saved skill |
| `!nilo skills` | List all skills |
| `!nilo forget <name>` | Delete a skill |
| `!nilo queue <task>` | Add task to autonomous curriculum |
| `!nilo autonomous on/off` | Toggle autonomous skill learning |

**LLM model:** `llama3.1:8b` by default (override with `NILO_CODE_MODEL` env var).  
**Autonomous gap:** 10 min between curriculum ticks; only runs if no player interaction in 10 min.

**Pre-built skills (handcrafted, always available):**  
`jump` `jump_5_times` `wave` `spin` `sprint_forward` `move_forward` `move_back` `move_left` `move_right` `look_around` `crouch_stand`

## Pathfinding Notes (`movement.js`)

- Uses `mineflayer-pathfinder`. `thinkingTimeout` set to 5000ms for complex modded terrain.
- **Door opener:** a `physicsTick` listener proactively opens adjacent closed doors (Fabric server fix — pathfinder's built-in opener misbehaves on non-Paper).
- **Fence set cleanup:** modded stone/brick blocks are removed from `movements.fences` to prevent false impassable-block classification.
- **Stuck recovery:** `path_reset` (reason=stuck) triggers `tryUnstuck`. Three consecutive stucks in the same spot → abandon path and pick a fresh random target.

## How to Add Things

### New natural language command
1. Add an `IS_*` matcher constant in `commands.js` (array of regexes, `cmd([...])` wrapper).
2. Add a handler block in `handleNaturalCommand` before `return false`.

### New LLM-dispatchable action
1. Add a `case 'name':` block in `dispatchAction` in `actions.js`.
2. Add `'name'` to the `actionHint` string in `nilo.js`.

### New pre-built skill
1. Create `skills/<name>.js` exporting `async function niloSkill(bot)`.
2. Add an entry to `skills/manifest.json`.

### New behavior mode
1. Add a mode string — no enum to update, just use it.
2. Call `setBehavior(bot, 'newmode', username)` to enter.
3. Check `state.behaviorMode === 'newmode'` inside your interval/loop.
4. `clearBehavior` will kill the interval automatically if stored in `state.behaviorInterval`.
